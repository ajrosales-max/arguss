Arguss needs a fine-grained PAT scoped only to the repository you want it to act on.

  • Repository access: "Only select repositories" → pick the target repo
  • Contents: Read and write (to push commits and create branches)
  • Pull requests: Read and write (to open PRs)
  • Actions: Read (to check CI status)

Generate a PAT on GitHub →

Arguss never stores your PAT. The token is held in memory for the duration of this single scan and discarded immediately afterward. It is not logged, not written to disk, and not included in any response or PR body.

By submitting a PAT you authorize Arguss to open pull requests on your behalf.
